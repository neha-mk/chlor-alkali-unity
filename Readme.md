# Chlor-Alkali Unity Project – Architecture Summary

## 1. Project Overview
- **Type:** Interactive Educational Simulation (Industrial Chemistry)
- **Purpose:** Visualize and simulate the chlor-alkali process (electrolysis) with real-time 3D interaction.

## 2. Application Structure
- **Scenes:**  
  - Welcome Scene (entry, info, navigation)  
  - Industrial View (main simulation, interactive equipment, particle systems, current control)  
  - Cell View (detailed analysis, metrics, info panels)  
- **Scene Transitions:** Welcome → Industrial View → Cell View

## 3. Core Components
- **SimulationController:** Main logic hub; manages current, particle systems, production metrics, and audio.
- **UIManager:** Singleton for info/legend/equations/formulas panels; ensures only one panel visible at a time.
- **CellClickController:** Handles camera transitions and scene navigation.
- **WelcomeManager:** Controls welcome scene UI and navigation.
- **ElectronSpawner/ElectronMover:** Spawns and animates electrons along a path.
- **InteractiveEquipment/EquipmentData:** Handles equipment hover/click, tooltips, and info display.
- **TooltipManager:** Singleton for tooltip rendering.
- **GuideManager:** Manages tutorial/guide steps.
- **CameraIntroOrbit/StartupSequence:** Camera intro animation and particle system startup sequencing.
- **UIEffects/SceneTracker:** UI animations and navigation state tracking.

## 4. Data Flow & Interactions
- **Simulation Loop:**  
  Slider → SimulationController → Particle Systems & Text Displays → Electron Spawner
- **User Interaction:**  
  Hover/Click Equipment → InteractiveEquipment → TooltipManager/UIManager
- **Scene Navigation:**  
  WelcomeManager/CellClickController manage transitions and camera movement.

## 5. Asset Structure
- **3D Models:** Industrial cell, tanks, pumps, pipes, etc. (FBX)
- **Materials:** For ions, gases, liquids, equipment, and effects.
- **Prefabs:** Electron.prefab for animated electron visualization.
- **UI Assets:** Fonts, images (legends, equations), audio.
- **Input:** Unity Input System (keyboard/mouse).

## 6. Technical Dependencies
- Unity packages: uGUI, Input System, URP, Cinemachine, Timeline, etc.
- Code uses UnityEngine, UnityEngine.UI, TMPro, SceneManagement, EventSystems.

## 7. Design Patterns
- **Singleton:** UIManager, GuideManager, TooltipManager, UIEffects.
- **Component:** EquipmentData + InteractiveEquipment.
- **Observer:** UI updates via slider events.
- **Dynamic Component Attachment:** ElectronMover added at runtime.
- **State Machine:** SceneTracker for navigation.
- **Factory (implicit):** ElectronSpawner.

## 8. Architectural Layers
- **Presentation:** UI panels, tooltips, legends (UIManager, TooltipManager, GuideManager)
- **Interaction:** User input (InteractiveEquipment, button handlers)
- **Simulation:** Business logic (SimulationController, ElectronSpawner)
- **Visualization:** Rendering (ParticleSystems, Materials, Models)

## 9. Configuration Points
- Inspector-editable parameters for controllers, spawners, and managers.
- Scene-specific tutorial steps and camera parameters.

## 10. Persistence & State
- SceneTracker for navigation history.
- No persistent save system or player preferences.

## 11. User Experience Flow
1. Launch → Welcome Scene (branding, info, start)
2. Start → Industrial View (camera intro, tutorial, simulation)
3. Interact: Adjust current, observe changes, explore equipment
4. Zoom in for Cell View (detailed analysis)
5. Return to menu as needed

## 12. Key Integration Points for Diagram
- Data flow: Slider → SimulationController → Particle Systems/Text
- User interaction: Equipment → InteractiveEquipment → UIManager/TooltipManager
- Scene hierarchy: Welcome ↔ Industrial View ↔ Cell View
- Initialization: StartupSequence
- Camera: CameraIntroOrbit ↔ CellClickController
- Tutorial: GuideManager
